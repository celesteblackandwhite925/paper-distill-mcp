# 研究总结生成指令

请根据我们今天的对话内容，生成一份结构化研究总结。我会把这份总结发给我的论文推送系统，它会自动帮我：
- 按知识主题归档到 Obsidian 研究笔记
- 生成当日学习日志
- 把提到的论文存入 Zotero

## 输出格式（严格遵守）

```markdown
# 研究总结 YYYY-MM-DD

## 主题 1：<知识点名称>

**领域**：<LLM / RAG / Agents / Code Generation / Multimodal / Diffusion / RL / 其他>
**标签**：#tag1 #tag2 #tag3

<这个主题我今天学到了什么，3-5 句话。写清楚核心概念、关键结论、我的理解。>

**要点**：
- 要点 1
- 要点 2

**相关论文**：
1. <论文标题>
   - 作者：<第一作者 et al.> | 期刊：<期刊名> | 年份：<YYYY>
   - DOI：<10.xxxx/xxxx>
   - 摘要：<这篇论文讲了什么、用了什么方法、得到什么结果，2-3 句>

---

## 主题 2：<知识点名称>

（同上格式）

---

## 新想法 / 新兴趣（如有）

- <随手想到的研究灵感、想探索的新方向>
```

## 规则

1. **主题是知识点**，不是论文标题
   - ✅ "Chain-of-Thought 推理策略"、"RAG 中的混合检索方法"、"多模态对齐训练"
   - ❌ "读了 Smith 2024 的论文"、"这篇 Nature 文章"

2. **同一知识点下的论文归到一起**
   - 如果我们讨论了 3 篇关于 RAG 检索的论文，它们都归到"RAG 检索"这一个主题下

3. **论文信息尽量完整**
   - DOI 是必须的，没有就写 `DOI：未知`
   - 作者至少写第一作者，不要写 Unknown
   - 如果对话中没提到具体论文，"相关论文"部分可以省略

4. **领域归类**
   - 按论文实际研究方向分类
   - 通用深度学习方法按其应用场景归类
   - 不确定的归到"其他"

5. **标签选择**
   - 方法标签：#深度学习 #transformer #attention #RLHF #RAG #chain-of-thought #diffusion #GNN 等
   - 领域标签：#LLM #reasoning #code-gen #multimodal #vision-language #agents 等
   - 每个主题 2-5 个标签

6. **语言**：中文，自然口语化，像给自己写的笔记

7. **不要遗漏**：对话中讨论过的每个实质性知识点都要覆盖到

8. **新想法**：如果对话中我提到了新的研究灵感或想探索的方向，单独列在最后
